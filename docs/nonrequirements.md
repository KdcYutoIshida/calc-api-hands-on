# 非機能要件（最小構成 / PoC）

## 実行環境 / リージョン
- Azure Functions v4（Python 3.12 系、開発は 3.12.10 を想定）
- プラン: Consumption Plan（コスト最小）
- リージョン: 東日本（Japan East）のみ。DR/フェイルオーバーなし。
- コールドスタートは許容。

## セキュリティ
- 認証レベル: Anonymous（APIキー不要）
- ネットワーク制約: なし（VNet統合やIP制限なし）
- HTTPS強制: 行わない（PoCの最低要件。利用時はHTTPS推奨）
- CORS: 設定不要

## 監視・ログ・運用
- 監視: 行わない（Application Insights 無効、アラートなし）
- ログ: プラットフォーム標準ログのみ（詳細な構造化ログ/相関IDは非対応）
- 運用: 手動デプロイ想定（CI/CDの整備は不要）

## 可用性・性能
- SLO/SLA: 規定しない（PoC）
- 性能目標: 規定しない。小規模アクセス前提。

## コスト
- Consumption Plan を採用し、追加リソースは作成しない（最小コスト）。
- 外部サービス連携や有料監視は利用しない。

## 開発プロセス
- 依存パッケージ: 追加なし（標準ライブラリのみ想定）
- コード規約/テスト: 必須要件としない（必要に応じて最小限）
- バージョニング: APIのバージョン付けは行わない（Breaking change 許容の PoC）
